## Hard Constraints

1. **Markdown is the source of truth.** The SQLite index is disposable and
   regenerated; never treat it as authoritative, never commit it.
2. **Never hand-edit generated views.** `JOURNAL.md` and `themes/*.md` in a
   managed journal are regenerated by `refresh_views()`. Changes go in entry
   files; views follow.
3. **No data loss in intake/migration paths.** Originals move to `attic/`,
   never deleted. Any new destructive-looking operation must preserve a
   recoverable copy and be covered by a test proving it.
4. **Indexed-mode journals are read-only.** Never restructure or write to a
   source configured with `mode = "indexed"`.
5. **No git staging or committing.** Never run *or suggest* `git add`,
   `git rm --cached`, `git restore --staged`, `git commit`, or `git push`.
   Present what changed (a file list or diff) and stop — Ken stages and commits.

## Working in This Repo

- Use the project venv. Prefer the make targets (they pin to `.venv/bin` and
  match CI): `make test`, `make check`, `make lint-fix`. Direct equivalents:
  `.venv/bin/pytest -q`, `.venv/bin/ai-journal ...`. Setup: `./scripts/setup.sh`.
  Full guide in `docs/DEVELOPMENT.md`.
- Before declaring work done, run `make check` (lint + format-check +
  type-check + full suite with coverage). It must be green.
- Ruff is the single linter/formatter and mypy the type authority; all tool
  config lives in `pyproject.toml`. Don't add competing config files.
- New behavior requires tests. Test data is fixture-based, not inline:
  - **Formats/parsing** → drop a real sample under `tests/fixtures/` and let
    `parametrize` pick it up (parser changes *must* add a fixture reproducing
    the real-world format that motivated them — see
    `tests/fixtures/parser/README.md`).
  - **Pure logic** → table-driven via `@pytest.mark.parametrize`.
  - **Filesystem behavior** → the `make_journal` factory fixture in
    `tests/conftest.py`, not ad-hoc inline file-writing.
- The MCP server (`server.py`) is a thin layer; logic belongs in the
  library modules so it stays testable without an MCP client.
- Ken's live journals are configured in `~/.config/ai-journal/journals.toml`.
  Do not run `migrate --apply` or bulk-edit scripts against them without an
  explicit request and a fresh backup.
